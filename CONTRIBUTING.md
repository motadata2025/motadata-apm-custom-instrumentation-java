# Contributing

Thank you for your interest in contributing to the Motadata APM Custom Instrumentation Java library!

## Proprietary Software

This project is significantly different from typical open-source projects. It is **Proprietary Software** owned by Motadata, and while we use GitHub for internal distribution and collaboration, it is not licensed for public contribution or unrestricted reuse.

### Reporting Issues

If you encounter bugs or have feature requests, please:
1.  Check the [Issue Tracker](https://github.com/motadata2025/motadata-apm-custom-instrumentation-java/issues) to see if it has already been reported.
2.  If not, create a new issue with a clear description, reproduction steps, and (if applicable) log files.

### Submitting Changes

**Only authorized Motadata engineers and contributors** may submit Pull Requests.
If you are an external user and wish to suggest a change:
*   Please open an Issue first to discuss your proposal.
*   Do not submit a PR without prior approval.

## Code Standards

If you are authorized to contribute, please adhere to:
*   **Java Version**: JDK 8 compatibility.
*   **Style**: Allman braces (braces on new lines).
*   **Exceptions**: Use checked exceptions (`Exception`) for all user-facing methods.
*   **Dependencies**: No new external dependencies without architectural review.
